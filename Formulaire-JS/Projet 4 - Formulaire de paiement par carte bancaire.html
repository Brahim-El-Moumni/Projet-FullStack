Explication de la page

Cette page HTML contient donc un formulaire de paiement. Il y a une div parent avec la classe “container” qui contient tous les éléments suivants :

Une div avec la classe “contMontant” qui contient un paragraphe avec le texte “1 Mug blanc : 5€”.

Un formulaire qui commence avec la balise <form>.

Dans le formulaire, il y a trois divs contenant des étiquettes et des champs d’entrée pour les informations du titulaire de carte, les numéros de carte, la date d’expiration et le cryptogramme.

La première div a la classe “inpFull” et contient une étiquette pour le champ d’entrée “Titulaire” et un champ d’entrée de type “text” avec un id “nom”, un nom “titulaire”, un attribut “required” et un attribut “maxlength” de 30 caractères.

Il y a également un attribut “placeholder” pour donner des instructions de saisie à l’utilisateur.

La deuxième div a également la classe “inpFull” et contient une étiquette pour le champ d’entrée “Numéros” et un champ d’entrée de type “text” avec un id “num”, un nom “numeros”, un attribut “required”, un attribut “pattern” pour n’accepter que des chiffres et des espaces et un attribut “placeholder” pour les numéros de carte.

La troisième div a la classe “contExSecu” et contient deux divs enfants. La première div a la classe “inpExp” et contient une étiquette pour le champ d’entrée “Expiration”, deux sélecteurs de liste déroulante pour le mois et l’année d’expiration et un span qui contient le caractère “/”.

La deuxième div a la classe “inpSecu” et contient une étiquette pour le champ d’entrée “Cryptogramme” et un champ d’entrée de type “text” avec un nom “crypto”, un id “crypto”, un attribut “required”, un attribut “maxlength” de 3 caractères et un attribut “pattern” pour n’accepter que des chiffres.

La dernière div a la classe “contBtn” et contient un bouton de type “submit” avec le texte “Confirmer et payer”.

display: block; : Le bouton est affiché comme un élément de type “bloc”, ce qui signifie qu’il occupe toute la largeur de son conteneur et crée une nouvelle ligne.

position: absolute; : Le bouton est positionné absolument par rapport à son premier parent positionné. Cela signifie que les propriétés top, left, right et bottom peuvent être utilisées pour définir sa position exacte.

font-size: 18px; : Définit la taille de police du texte à 18 pixels.

transition: 0.2s ease background; : Cette propriété crée une transition en douceur de 0,2 seconde lorsqu’on survole le bouton. L’effet de transition affecte uniquement la propriété background (couleur de fond).

Réalisation du Javascript

Dans ce projet nous n’allons pas directement ajouter notre script JS dans une balise <scrip>...</scrip> directement dans notre page HTML mais nous allons faire appel à un script app.js que nous appelerons dans notre page.

Du coup dans notre page HTML nous devons ajouter l’appel à notre script en toute fin du body.













<html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Carte de crédit</title>
    <link rel="stylesheet" href="style.css">
  </head>
  <body>
    ...
    <script>app.js</script>
</body>
</html>

Passons maintenant au script JS.


















const inpCard = document.getElementById('num');

inpCard.addEventListener('keyup', () => {
    
    let val = inpCard.value;
    let newval = '';

    val = val.replace(/\s/g, '');
    
    for(let i = 0; i < val.length; i++) {
        if(i % 4 === 0 && i > 0) {
            newval = newval.concat(' ');
        }
        newval = newval.concat(val[i]);      
    }
    inpCard.value = newval;
})

Ici dans notre JavaScript utilise un écouteur d’événement pour suivre l’événement “keyup” sur un élément de formulaire HTML avec l’ID “num”. Lorsque l’utilisateur saisit quelque chose dans cet élément, la fonction fléchée associée à l’événement est appelée.

La fonction récupère la valeur actuelle de l’élément “num” en utilisant la propriété value de l’objet input. Ensuite, elle utilise la méthode replace() avec une expression régulière pour supprimer tous les espaces dans la chaîne de caractères.

Ensuite, la fonction boucle à travers chaque caractère de la chaîne de caractères de l’input. Si l’indice est divisible par 4 et est supérieur à 0, cela signifie qu’il faut ajouter un espace avant le caractère. Cette opération ajoute un espace tous les 4 caractères pour simuler le formatage d’un numéro de carte de crédit.

Finalement, la fonction met à jour la valeur de l’input avec la nouvelle valeur formatée. Ainsi, chaque fois que l’utilisateur tape dans l’input, le numéro de carte de crédit est automatiquement formaté pour ressembler à un numéro de carte valide.
