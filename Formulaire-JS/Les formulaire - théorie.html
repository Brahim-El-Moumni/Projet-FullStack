Les formulaire - théorie

Introduction

Dans ce chapitre nous allons revoir les formulaires que nous avons déjà vu dans le coours sur le HTML.

Les chapitres de ce cours seront donc de simples rappel à mise en oeuvre des formulaire en HTML et les différentes balises que l’on peut utiliser à cet effet.

Nous verrons par la suite comment les rendres plus attractif en javascript lorsque nous réaliserons les différents cas pratique.

Rappel sur les formulaires

Formulaire classique :

Ici nous verrons comment faire un formulaire classique, basique sans CSS ni rien et dans lequel on va juste faire la sturcture HTML.

Pour rappel pour réaliser un formulaire nous devons l’entourer d’une balise <form>




<form>
    ...
</form>

Les champs input :

Donc nous allons à l’intérieur de cette balise form ajouter des input qui vont nous servir pour recuillir les données. La balise <input> est une balise qui n’a pas besoin d’être fermé. Elle peut prendre plusieurs attribut, notamment un type qui peut être un type texte, qui est le type de base qui va nous permettre d’entrer du texte à notre formulaire.




<form>
    <input type="text">
</form>

Le bouton :

Ensuite, une fois que nous avons notre formulaire nosu avons besoin d’un bouton nous permettant de valider ou d’envoyer les données de notre formulaire.

Pour créer un bouton pour les formulaires nous avons deux possibilité :

  1. Utilisation de la balise <button>...</button>.
  2. Utilisation d’un input avec comme type submit.

les deux façon de faire son totalement similaire l’une n’ets pas meilleur que l’autre.

Cas du bouton





<form>
    <input type="text">
    <button>Envoyer</button>
</form>

Le cas avec un input





<form>
    <input type="text">
    <input type="submit" value="Envoyer">
</form>

Donc ici nous avons notre premier formulaire de fortune.

Le formulaire va automatique détecter lorsque l’utilisateur clique sur un bouton dans un formulaire qu’il souhaite envoyer les données du formulaire.

Donc pour rappel un formulaire c’est avant tout une balise englobante, dans laquel on retrouvera des champs, qui peuvent être de type “text” ou autre et à la fin un bouton permettant d’envoyer les données.

les différents type d’input

Nous venons donc de voir l’input de type “text”. Il existe d’autre type de champs pour les formulaire et nous allons, dans cette section, en voir quelques un.

Le type email





<form>
    <input type="email">
    <input type="submit" value="Envoyer">
</form>

Le type de champ email est lun type plutôt interrassant qui va permettre de définir un champ spécifique pour recevoir des adresses email. Ce type champ va nous permettre d’avoir une première validation sur le champ si l’entrée est un email syntaxiquement valide.

Le type password





<form>
    <input type="password">
    <input type="submit" value="Envoyer">
</form>

Le champ password est utilisé pour saisir un mot de passe confidentiel qui est masqué à l’utilisateur.

le type search





<form>
    <input type="search">
    <input type="submit" value="Envoyer">
</form>

Le type “search” est un type de champ de formulaire HTML qui permet à l’utilisateur de saisir une requête de recherche. Il est souvent utilisé dans les formulaires de recherche sur les sites web.

Lorsqu’un utilisateur saisit une requête dans un champ de recherche de type “search”, le navigateur affiche souvent une petite croix permettant de supprimer l’entré ou une petite loupe à côté du champ pour indiquer qu’il s’agit d’un champ de recherche.

tel





<form>
    <input type="tel">
    <input type="submit" value="Envoyer">
</form>

Le type “tel” est utilisé pour demander à l’utilisateur de saisir un numéro de téléphone. Ce champ peut aider à valider les numéros de téléphone et à assurer que les utilisateurs saisissent des numéros de téléphone valides. Les navigateurs mobiles utilisent souvent un clavier numérique lorsqu’un champ “tel” est affiché à l’écran.

Le comportement par défaut de ce type de champ peut varier selon les navigateurs et les systèmes d’exploitation, mais il est souvent associé à des fonctionnalités supplémentaires, telles que la possibilité de passer un appel téléphonique en cliquant sur un bouton ou la possibilité d’ajouter le numéro de téléphone dans les contacts de l’utilisateur

Le type url





<form>
    <input type="url">
    <input type="submit" value="Envoyer">
</form>

Le type “url” est un type de champ de formulaire en HTML qui permet à l’utilisateur de saisir une URL (Uniform Resource Locator). Le champ de saisie est validé automatiquement pour s’assurer qu’il contient une URL valide, c’est-à-dire une adresse web commençant par http:// ou https://.

Ce type de champ est souvent utilisé dans les formulaires de contact ou de commentaire pour permettre aux utilisateurs de saisir des liens vers des pages web pertinentes.

Le type date





<form>
    <input type="date">
    <input type="submit" value="Envoyer">
</form>

Le type de champ “date” est utilisé dans les formulaires HTML pour permettre aux utilisateurs de saisir des dates.

Ce champ affiche un calendrier qui permet aux utilisateurs de sélectionner facilement une date à partir de laquelle les données sont entrées.

Le format de la date peut être spécifié à l’aide de l’attribut “format” et l’attribut “min” et “max” peut être utilisé pour définir une plage de dates valides.

Le type month





<form>
    <input type="month">
    <input type="submit" value="Envoyer">
</form>

Le type “month” est un type de champ pour les formulaires HTML qui permet à l’utilisateur de saisir un mois et une année, sans indication de jour.

Le champ permet d’afficher un calendrier déroulant avec des options pour le mois et l’année.

Lorsqu’une valeur est soumise, elle est envoyée au serveur sous forme d’une chaîne de caractères au format “AAAA-MM”, où AAAA est l’année et MM est le mois (par exemple, “2023-03” pour mars 2023).

Le type number





<form>
    <input type="number">
    <input type="submit" value="Envoyer">
</form>

Le champ number est utilisé pour saisir une valeur numérique. Il peut être configuré pour spécifier une plage de valeurs autorisées, une valeur par défaut, etc. le type number peut prendre des attributs tel que min et max pour définir les valeurs minimal et maximal du champ.





<form>
    <input type="number" min="1" max="10">
    <input type="submit" value="Envoyer">
</form>

Le type range





<form>
    <input type="range" min="0" max="100" value="50">
    <input type="submit" value="Envoyer">
</form>

Permet de sélectionner une valeur numérique dans une plage donnée, généralement en utilisant un curseur ou une barre de défilement.

Voici un exemple de code HTML pour créer un champ de formulaire de type “range” avec une valeur minimale de 0 et une valeur maximale de 100, et une valeur initiale de 50 :

<input type="range" id="rangeInput" name="rangeInput" min="0" max="100" value="50">

Lorsque l’utilisateur interagit avec le champ de formulaire, la valeur sélectionnée sera renvoyée sous forme de nombre entier entre la valeur minimale et la valeur maximale. Dans cet exemple, la valeur sélectionnée peut être n’importe quel nombre entier entre 0 et 100.

Un peut de javascript

Arrêtons nous un peut sur ce type pour faire un peu de JS afin de nous permettre d’afficher la valeur numérique de notre champ.















<form>
    <input type="range" id="rangeInput" min="0" max="100" value="50">
    <input type="submit" value="Envoyer">
</form>
<p id="resultat"></p>

<script>
    let resultat = document.getElementById("resultat");
    let rangeInput = document.getElementById("rangeInput");
    
    rangeinput.oninput = function() {
        resultat.textContent = rangeInput.value;
    }
</script>

Le code HTML crée un formulaire avec un champ de type “range” et un bouton de soumission. Le champ de type “range” permet de sélectionner une valeur numérique dans un intervalle donné (ici de 0 à 100 avec une valeur par défaut de 50).

Le code JavaScript récupère les éléments HTML avec les identifiants “resultat” et “rangeInput” en utilisant la méthode getElementById().

Il crée ensuite un gestionnaire d’événements qui se déclenche lorsque la valeur du champ “rangeInput” change.

À chaque changement de valeur, la fonction affectée à la propriété oninput du champ “rangeInput” met à jour le contenu de l’élément HTML “resultat” avec la nouvelle valeur du champ “rangeInput”.

Ainsi, lorsque l’utilisateur déplace le curseur du champ “rangeInput”, la nouvelle valeur sélectionnée est affichée en temps réel dans l’élément HTML “resultat”.

Les labels
parton du formulaire rudimentaire suivant :






<form>
    <input type="text">
    <br><br>
    <input type="submit" value="Envoyer">
</form>

Les labels de formulaire en HTML sont des éléments qui permettent d’associer un libellé à un champ de formulaire, comme un champ de texte, un bouton radio ou une case à cocher. Les labels améliorent l’accessibilité des formulaires en permettant aux utilisateurs de cliquer sur le libellé pour sélectionner le champ de formulaire associé, plutôt que de devoir cliquer précisément sur le champ de formulaire lui-même.

Pour créer un label, on utilise l’élément HTML <label> et on l’associe au champ de formulaire avec l’attribut for qui a pour valeur l’identifiant unique du champ.

Voici un exemple :







<form>
    <label for="nom">Nom :</label>
    <input type="text" id="nom" name="nom">
    <br><br>
    <input type="submit" value="Envoyer">
</form>

Dans cet exemple, le label “Nom :” est associé au champ de texte grâce à l’attribut for=“nom”, qui correspond à l’attribut id=“nom” du champ de texte.

Les attributs des inputs

l’attribut name







<form>
    <label for="nom">Nom :</label>
    <input type="text" id="nom" name="nom">
    <br><br>
    <input type="submit" value="Envoyer">
</form>

L’attribut name d’un élément input en HTML est utilisé pour spécifier le nom de l’élément qui sera envoyé avec le formulaire lors de la soumission. Lorsqu’un formulaire est soumis, les valeurs des éléments de formulaire sont envoyées au serveur sous forme de paires clé-valeur. Le nom de l’élément est utilisé comme clé et la valeur saisie dans l’élément est utilisée comme valeur.

L’attribut name est également utilisé pour lier un label à son élément de formulaire associé. En associant le name de l’élément avec le for de l’élément label, un clic sur le label provoque le focus de l’élément de formulaire associé.

L’attribue value







<form>
    <label for="nom">Nom :</label>
    <input type="text" id="nom" name="nom" value="Jhon">
    <br><br>
    <input type="submit" value="Envoyer">
</form>

L’attribut value dans la balise input est utilisé pour spécifier la valeur initiale d’un champ de formulaire. Lorsqu’un utilisateur soumet un formulaire, la valeur de l’attribut value sera incluse dans les données envoyées au serveur.

Par exemple, si vous avez un champ de formulaire de type text comme celui-ci :

<input type="text" name="username" value="John">

Lorsque le formulaire est soumis, la valeur “John” sera envoyée au serveur en même temps que le nom de l’entrée “username”.

L’attribut value peut être utilisé avec plusieurs types de champs de formulaire, tels que text, password, radio, checkbox, hidden, button, submit, reset, image, file, date, color, etc.

Notez que si un utilisateur modifie la valeur d’un champ de formulaire, la valeur de l’attribut value ne sera pas mise à jour automatiquement. Si vous voulez récupérer la valeur actuelle d’un champ de formulaire, vous devez utiliser le DOM pour accéder à la propriété value de l’élément.

L’attribut placeholder







<form>
    <label for="nom">Nom :</label>
    <input type="text" id="nom" name="nom" value="Jhon" placeholder="Votre nom ici">
    <br><br>
    <input type="submit" value="Envoyer">
</form>

L’attribut placeholder d’un élément input est utilisé pour afficher un texte d’exemple dans l’élément de formulaire lorsqu’aucune valeur n’est saisie.

Il est souvent utilisé pour donner des indications aux utilisateurs sur le format ou le type de données attendus dans l’élément de formulaire.

Par exemple, si vous avez un champ de saisie d’adresse e-mail, vous pouvez utiliser l’attribut placeholder pour donner un exemple de format d’e-mail, tel que “votre_email@example.com”.

Voici un exemple d’utilisation de l’attribut placeholder pour un champ de saisie d’adresse e-mail :

<input type="email" name="email" placeholder="Entrez votre adresse e-mail">

Textarea, select et datalist

textarea

<textarea> est une balise HTML qui permet de créer un champ de texte multi-lignes dans un formulaire. Il est principalement utilisé pour permettre aux utilisateurs de saisir des commentaires, des messages ou d’autres types de données textuelles.

La syntaxe de base pour créer un champ textarea est la suivante :

<textarea name="nom-du-champ" rows="nombre-de-lignes" cols="nombre-de-colonnes">valeur-par-défaut</textarea>

L’attribut name définit le nom du champ qui sera envoyé au serveur lors de la soumission du formulaire.
Les attributs rows et cols spécifient respectivement le nombre de lignes et de colonnes que le champ textarea doit afficher.
La valeur par défaut du champ textarea peut être définie entre les balises <textarea> et </textarea>.
Exemple :

<textarea name="message" rows="4" cols="50" placeholder="Saisissez votre message ici"></textarea>

Dans cet exemple, nous avons créé un champ textarea nommé “message” qui affiche 4 lignes et 50 colonnes et qui a un texte d’aide de saisie (placeholder) “Saisissez votre message ici”.

L’élément select

<select> est une balise HTML qui permet de créer une liste déroulante (dropdown) dans un formulaire. Elle est souvent utilisée pour les menus déroulants ou pour les listes de choix dans les formulaires. Les options de la liste sont définies par des balises <option> qui sont des enfants de la balise <select>.

Exemple :









<label for="pays">Pays :</label>
<select id="pays" name="pays">
  <option value="france">France</option>
  <option value="espagne">Espagne</option>
  <option value="italie">Italie</option>
  <option value="portugal">Portugal</option>
  <option value="allemagne">Allemagne</option>
</select>

Dans cet exemple, la balise <select> définit une liste déroulante pour les pays et chaque option est définie par une balise <option> avec la valeur correspondante dans l’attribut value et le texte affiché dans l’option entre les balises ouvrantes et fermantes.

On peut ajouter dans l’une des optiosn d’un élément select l’attribut selected, pour que cette option soit sélectionné par défaut.

l’élément optgroup de select

L’élément optgroup est utilisé dans un élément select pour grouper les options en catégories. Cela permet d’organiser les options en sous-groupes logiques, ce qui peut faciliter la sélection de l’utilisateur.

L’élément optgroup doit être placé à l’intérieur d’un élément select et doit contenir un ou plusieurs éléments option. Il peut également avoir un attribut label qui spécifie le nom du groupe.

Voici un exemple d’utilisation de optgroup :














<label for="fruits">Choisissez un fruit :</label>
<select id="fruits">
  <optgroup label="Fruits rouges">
    <option value="fraise">Fraise</option>
    <option value="framboise">Framboise</option>
    <option value="cerise">Cerise</option>
  </optgroup>
  <optgroup label="Fruits jaunes">
    <option value="banane">Banane</option>
    <option value="ananas">Ananas</option>
    <option value="mangue">Mangue</option>
  </optgroup>
</select>

Dans cet exemple, les options sont organisées en deux groupes, “Fruits rouges” et “Fruits jaunes”, pour aider l’utilisateur à sélectionner le fruit qu’il recherche plus facilement.

Les datalist
<datalist> est une balise HTML qui permet de proposer une liste de choix pour un champ de formulaire, mais contrairement à <select>, la liste n’est pas déroulante. Elle est souvent utilisée pour les champs de recherche ou de suggestion. Les options de la liste sont définies par des balises <option> qui sont des enfants de la balise <datalist>.

Exemple :










<label for="ville">Ville :</label>
<input type="text" id="ville" name="ville" list="villes">
<datalist id="villes">
  <option value="Paris">
  <option value="Lyon">
  <option value="Marseille">
  <option value="Toulouse">
  <option value="Bordeaux">
</datalist>

Dans cet exemple, la balise <input> définit un champ de texte pour la ville avec un attribut list qui fait référence à la balise <datalist> qui contient les options de la liste de choix. Les options sont définies par des balises <option> avec la valeur correspondante dans l’attribut value. Lorsque l’utilisateur commence à taper dans le champ de texte, une liste de suggestions basée sur les options de la balise <datalist> apparaît.

Checkbox, radio et color

Les checkboxes, les radios et les inputs de type “color” sont tous des types d’éléments de formulaire en HTML.

Checkbox : Une checkbox permet à l’utilisateur de sélectionner une ou plusieurs options à partir d’un ensemble d’options. Elle est représentée par une case à cocher. Lorsqu’une case est cochée, la valeur correspondante est envoyée lors de la soumission du formulaire. Elle est créée en utilisant l’élément <input> avec un attribut type="checkbox".

Exemple :






<label for="fruits">Quels fruits aimez-vous ?</label>
<input type="checkbox" name="fruits" id="orange" value="orange">
<label for="orange">Orange</label>
<input type="checkbox" name="fruits" id="pomme" value="pomme">
<label for="pomme">Pomme</label>

Radio : Les boutons radio permettent à l’utilisateur de sélectionner une seule option à partir d’un ensemble d’options. Chaque bouton radio doit avoir le même nom, mais une valeur différente. Elle est créée en utilisant l’élément <input> avec un attribut type="radio".
Exemple :






<label for="sexe">Quel est votre sexe ?</label>
<input type="radio" name="sexe" id="homme" value="homme">
<label for="homme">Homme</label>
<input type="radio" name="sexe" id="femme" value="femme">
<label for="femme">Femme</label>

Color : L’input de type “color” permet à l’utilisateur de choisir une couleur à partir d’une palette de couleurs. Elle est créée en utilisant l’élément <input> avec un attribut type="color".

Exemple :



<label for="couleur">Choisissez une couleur :</label>
<input type="color" id="couleur" name="couleur" value="#ff0000">

Notez que les valeurs de ces éléments de formulaire seront envoyées dans le corps de la requête HTTP lors de la soumission du formulaire.

Les champs actions et method

Les attributs action et method sont utilisés pour définir où et comment envoyer les données d’un formulaire HTML lorsqu’il est soumis.

L’attribut action est utilisé pour spécifier l’URL de destination où les données du formulaire doivent être envoyées. Par exemple, si vous souhaitez envoyer les données à un fichier PHP nommé “traitement.php”, vous pouvez définir l’attribut action de la balise <form> comme suit :



<form action="traitement.php">

L’attribut method est utilisé pour spécifier la méthode HTTP à utiliser lors de l’envoi des données du formulaire. La méthode par défaut est GET, mais il est souvent préférable d’utiliser POST, car elle est plus sûre et plus adaptée à l’envoi de données sensibles.

Voici un exemple de formulaire avec les attributs action et method :












<form action="traitement.php" method="post">
  <label>
    Nom :
    <input type="text" name="nom">
  </label>
  <label>
    Prénom :
    <input type="text" name="prenom">
  </label>
  <button type="submit">Envoyer</button>
</form>

Dans cet exemple, les données du formulaire seront envoyées à la page “traitement.php” avec la méthode HTTP POST lorsque l’utilisateur soumettra le formulaire en cliquant sur le bouton “Envoyer”.

Les méthode GET et POST

Les requêtes GET et POST sont les deux méthodes HTTP les plus couramment utilisées pour envoyer des données à partir d’un formulaire HTML.

La méthode GET est utilisée pour demander des ressources au serveur en envoyant des paramètres dans l’URL de la requête. Les données sont visibles dans l’URL, donc elle n’est pas recommandée pour l’envoi de données sensibles telles que les mots de passe.

Exemple de formulaire GET:





<form action="search.php" method="get">
  <input type="text" name="query">
  <input type="submit" value="Rechercher">
</form>

La méthode POST, quant à elle, est utilisée pour envoyer des données au serveur dans le corps de la requête HTTP. Les données ne sont pas visibles dans l’URL, donc elle est plus sûre pour l’envoi de données sensibles.

Exemple de formulaire POST:










<form action="signup.php" method="post">
  <label>Nom d'utilisateur:</label>
  <input type="text" name="username">
  <br>
  <label>Mot de passe:</label>
  <input type="password" name="password">
  <br>
  <input type="submit" value="S'inscrire">
</form>

Il est important de noter que les requêtes GET sont plus rapides à traiter et plus simples à utiliser, tandis que les requêtes POST sont plus sûres pour l’envoi de données sensibles et peuvent envoyer plus de données que les requêtes GET. Dans la plupart des cas, la méthode GET est utilisée pour récupérer des ressources, tandis que la méthode POST est utilisée pour envoyer des données à un serveur.

La validation

Maintenant que nous avons toute la partie input et tout les champs pour nos formulaire, nous allons à présent voir comment faire de la validation de données sur ceci.

Dans cette section nous verrons comment gérer par exemple le nombre de caractères que l’on peut mettre dasn un input ou dans un mot de passe par exemple, etc…

Pour illustrer tout cela nous allons créer un petit formulaire.





<form>
    <input type="email">
    <button>Envoyer</button>
</form>

Ici nous avons un input de type email. On peut observer que ce type de champ est automatiquement valider par le navigateur. Par exemple si l’utilisateur entre du texte qui n’a pas la syntaxe d’un email alors un message s’affiche lorsqu’on tente de valider le formulaire.

Le type de champ number aussi propose une validation qui va empêcher l’utilisateur d’écrire du texte dans le champ mais uniquement des valeurs numérique.

Pour le type “email”

required : le champ est obligatoire

Pour le type “password”

required : le champ est obligatoire
minlength : nombre minimum de caractères autorisés
maxlength : nombre maximum de caractères autorisés

Cas d’utilisation pour chaque type

required : Cet attribut permet de rendre obligatoire le remplissage d’un champ. Si l’utilisateur essaie de soumettre le formulaire sans remplir ce champ, une erreur sera affichée.



<label for="nom">Nom :</label>
<input type="text" name="nom" id="nom" required>

pattern : cet attribut permet de définir une expression régulière qui sera utilisée pour valider le contenu d’un champ. Si le contenu du champ ne correspond pas à l’expression régulière, une erreur sera affichée.



<label for="email">Adresse e-mail :</label>
<input type="email" name="email" id="email" required pattern="[a-z0-9._%+-]+@[a-z0-9.-]+\.[a-z]{2,}$">

Autre exemple de pattern



<label for="password">Mot de passe:</label>
<input type="password" id="password" name="password" pattern="^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)[a-zA-Z\d]{8,}$" required>

Dans cet exemple, l’attribut pattern est utilisé pour définir une expression régulière qui décrit les caractères autorisés pour le mot de passe. L’expression régulière utilisée est ^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)[a-zA-Z\d]{8,}$, qui exige que le mot de passe contienne au moins une lettre minuscule, une lettre majuscule et un chiffre, et qu’il ait une longueur d’au moins 8 caractères.

L’attribut required est également utilisé pour indiquer que ce champ est obligatoire et doit être rempli avant de soumettre le formulaire.

min et max : ces attributs permettent de définir des valeurs minimales et maximales pour un champ numérique.



<label for="age">Âge :</label>
<input type="number" name="age" id="age" min="18" max="99" required>

minlength et maxlength : ces attributs permettent de définir des longueurs minimales et maximales pour un champ de texte.



<label for="commentaire">Commentaire :</label>
<textarea name="commentaire" id="commentaire" rows="5" cols="30" minlength="10" maxlength="1000"></textarea>

step : cet attribut permet de définir un intervalle de valeurs pour un champ numérique.



<label for="prix">Prix :</label>
<input type="number" name="prix" id="prix" min="0" step="0.01" required>

autocomplete : cet attribut permet de définir si le navigateur doit ou non proposer des suggestions de saisie pour un champ.



<label for="adresse">Adresse :</label>
<input type="text" name="adresse" id="adresse" autocomplete="street-address">

list : cet attribut permet de lier un champ à une liste d’options définie ailleurs dans le formulaire.









<label for="fruit">Fruit préféré :</label>
<input type="text" name="fruit" id="fruit" list="fruits-list">
<datalist id="fruits-list">
  <option value="Pomme">
  <option value="Banane">
  <option value="Orange">
  <option value="Poire">
</datalist>

multiple : cet attribut permet de permettre la sélection de plusieurs options dans une liste déroulante ou une liste à choix multiple.








<label for="couleur">Couleur(s) préférée(s) :</label>
<select name="couleur" id="couleur" multiple>
  <option value="rouge">Rouge</option>
  <option value="vert">Vert</option>
  <option value="bleu">Bleu</option>
  <option value="jaune">Jaune</option>
</select>

petites astuces :

Lorsque des attribut de validation son positionné sur un champ input cela donne accès en CSS à deux pseudos classe qui sont la classe valid et invalid.

Sur ces pseudo classe nous pouvons très bien appliqué un style. Par exemple :








input:invalid {
    border: solid 1px red;
}

input:valid {
    border: solid 1px green;
}

Ce qui aura pour effet de de donner un contour rouge au champ qui n’on pas été validé et un contour vert au champ qui sont valide.

On peut aussi utiliser la propriété outline qui va nous permettre de retire le contour surbrillant (bleu en général) lorsqu’on clique sur le champ.




input {
    outline: none;
}

Il est aussi possible de changer le texte à l’interieur de la boite de dialogue qui apparait lorsque le champ n’est pas valide.

Prenons le cas d’un champ text simple avec un attribut required et un pattern qui impose des caractères de a à z en minuscule et des chiffres et une taille max de 6 caractères.

Ce qui nous donnerait le formulaire suivant :





<form>
    <input type="text" id="inpTxt" required pattern="[a-z0-9]{6,}">
    <button>Valider</button>
</form>

Lorsque l’on met des caractères non valide dans notre champ input ici, nous avons un message qui s’affiche dans une boite de dialogue avec le message suivant :

Veuillez respecter le format requis.

Nous pouvons décider de changer le message qui s’affiche. Pour cela nous pouvons utiliser le Javascript.







<script>
    cont monInp = document.getElementByid('inpTxt');
    
    monInp.setCustomValidity('Mon message perso');
    
</script>

Ce qui affichera à la place du message précédent le nouveau message suivant en cas d’entré de données non valide de l’utilisateur.
